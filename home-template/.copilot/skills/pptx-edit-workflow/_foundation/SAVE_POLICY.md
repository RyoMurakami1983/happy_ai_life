# 保存ポリシー

PPTX の編集では、見た目が正しくても後から崩れが見つかることがあります。  
そのため、この skill では **別名保存を既定** にします。

## 方針

- 原本は残す
- 変更後は新しいファイル名で保存する
- 上書き保存は例外として扱い、明示的な確認がある場合だけ行う
- 既定の保存先はローカルの作業領域に置く
- 共有・公開・同期フォルダへは明示確認なしで保存しない

## 命名の考え方

- 元ファイル名に用途を足す
- 版番号や日付を足して区別する
- 迷う場合は短く分かる名前を優先する

例:
- `proposal_reviewed.pptx`
- `proposal_v2.pptx`
- `deck_2026-04-05.pptx`

## 例外

上書きを選ぶなら、次の条件をそろえる。
- 原本の退避先がある
- 失敗時に戻せる
- その操作が意図的だと確認できる

## 補足

一時ファイルや作業用ファイルは、完了後に残さない。  
保存の安全性は、編集の正確さと同じくらい重要に扱う。
