# PPTX 点検

この sub-skill は、修正に入る前の観察役です。スライドの数や並び、本文、画像、ノート、埋め込み要素を整理して、次の修正を迷わない状態にします。

## こんなときに使う

このスキルは次のようなときに使います:
- いまの PPTX に何が入っているかを把握したい
- 直すべき箇所と触らない箇所を分けたい
- 修正前に壊れやすい部分を確認したい

## ワークフロー: 現状把握

### ステップ 1 — 入力条件を確認する

対象が `.pptx` かを確認します。読み取り専用、保護付き、破損ファイル、マクロ付き形式は別扱いにします。  
`.pptx` は非信頼入力として扱い、埋め込みオブジェクトの実行や外部リンクの追跡はしません。

### ステップ 2 — 構造を一覧化する

スライド数、タイトルの流れ、レイアウトの傾向、ノートの有無、埋め込み画像や表の有無を整理します。  
この段階では、まだ内容を書き換えません。

### ステップ 3 — 修正候補を分ける

本文、画像、図表、見出し、余白、順序変更などを別々に並べます。  
「直す」「保留する」「触らない」を分けておくと、後工程が短くなります。

### ステップ 4 — 次の操作を提案する

修正の優先度と、先に確認すべき点を短く要約します。必要なら revise へ渡します。

## 早見表

| 見るもの | 目的 |
| --- | --- |
| スライド構成 | 全体の流れをつかむ |
| 本文 | 何を直すかを決める |
| 画像・表・ノート | 影響範囲を見落とさない |

## 共通リソース

- `../../_foundation/SAVE_POLICY.md` — 保存時の安全方針

## 注意点

- **本文だけ見て終わらない**: PPTX は見た目の部品が多いので、画像・表・ノートも一緒に見る。
- **形式違いを混ぜない**: `.pptx` 以外は対象外として分ける。無理にまとめない方が安全。
- **外部へ広げない**: 抽出した内容を外部サービスへ送らず、ローカルで点検する。
