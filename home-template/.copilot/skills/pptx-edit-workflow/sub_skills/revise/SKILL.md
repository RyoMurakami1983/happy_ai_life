# PPTX 修正

この sub-skill は、点検結果をもとに実際の変更を進める役目です。編集した結果は原本と分けて保存し、あとから差分と結果を追いやすくします。

## こんなときに使う

このスキルは次のようなときに使います:
- スライドの文面を直したい
- 画像、表、順序を入れ替えたい
- 編集後のファイル名を分けて残したい

## ワークフロー: 変更と保存

### ステップ 1 — 変更内容を確定する

inspect の結果を見て、直す対象を小さく分けます。  
inspect が未実施で対象が曖昧なら、先に inspect に戻ります。  
一度に広く触るより、変更単位を絞る方が事故を減らせます。

### ステップ 2 — 変更を適用する

本文、レイアウト、画像、図表、ノートのうち、必要なものだけを更新します。  
触らない部品は触らないまま残します。

### ステップ 3 — 別名で保存する

保存は別名保存を既定にします。原本の上書きは、明示的な確認がある場合だけにします。  
保存先は原本と同じ信頼境界のローカル領域を既定にし、共有・公開・同期先へは明示確認なしで出しません。  
保存先の候補名は、元名に用途や版情報を足して区別しやすくします。

### ステップ 4 — 結果を確認する

開けること、見た目が崩れていないこと、置換漏れがないことを確認します。  
必要なら inspect に戻って再確認します。

## 早見表

| 状況 | 取る行動 |
| --- | --- |
| 修正対象が曖昧 | inspect に戻る |
| 変更が広い | 小さく分けて順番に直す |
| 保存先が決まっていない | SAVE_POLICY に沿って名前を決める |

## 共通リソース

- `../../_foundation/SAVE_POLICY.md` — 別名保存の方針

## 注意点

- **原本を守る**: 既定は別名保存。上書きは例外として扱う。
- **機密を外へ出さない**: PPTX の中身は外部サービスへ送らず、ローカルで扱う。
- **埋め込み要素を開かない**: 外部参照や埋め込みオブジェクトは実行・追跡しない。
