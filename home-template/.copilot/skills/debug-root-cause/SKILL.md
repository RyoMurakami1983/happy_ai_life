# debug-root-cause

この skill は、根本原因ベースで不具合を調査するための手順です。

## 目的
- 症状ではなく原因を修正する
- 再現条件を明確にする
- 将来の再発防止につなげる

## 手順
1. 症状を整理する。
   - 何が起きているか
   - 期待値は何か
   - いつから起きたか
2. 再現手順を作る。
   - 入力
   - 前提状態
   - 実行条件
3. 変更履歴、最近の差分、設定差異を確認する。
4. 問題領域を二分探索的に絞り込む。
5. 仮説を 1 つずつ検証する。
6. 応急処置と恒久対策を区別する。
7. 修正後、再現手順で再確認し、必要ならテスト化する。

## 観点
- 入力値の異常
- null / None / 未初期化
- 境界値
- 時刻 / タイムゾーン
- 並行実行
- 設定差異
- 依存ライブラリ更新
- 外部 API / I/O の失敗
- エラーハンドリングの握りつぶし

## 出力形式
1. 症状
2. 再現条件
3. 調査ログ
4. 原因候補
5. 最有力の根本原因
6. 修正案
7. 再発防止策
