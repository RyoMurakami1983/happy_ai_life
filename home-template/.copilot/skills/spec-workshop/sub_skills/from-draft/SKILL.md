# From Draft

既存の草案を仕様へ整える sub-skill です。ドラフトの足りない論点を対話で埋め、必要な事実だけ targeted research で補強します。

## こんなときに使う

このスキルは次のようなときに使います:
- ひな形やメモがすでにあるとき
- 草案の抜け漏れを埋めたいとき
- 仕様の確度を上げてから後段へ渡したいとき

## ワークフロー: ドラフトを仕様へ整える

### ステップ 1 — 草案を読む

既存のメモ、箇条書き、断片的な文章を読み、決まっていることと未決定を分けます。先に現物を読むことで、既にある判断を壊さずに済みます。

### ステップ 2 — 抜けている論点を問う

対話で、目的、範囲、例外、受け入れ条件の抜けを埋めます。質問は少なく、次の判断につながるものだけに絞ります。

### ステップ 3 — 必要な根拠を補う

調査が必要な論点だけ built-in 機能または自分の調査で補います。ドラフト全体を調べ直すのではなく、未確定箇所にだけ根拠を足します。

## 早見表

| 状況 | 次の動き |
| --- | --- |
| 草案に穴がある | 対話で論点を補う |
| 事実確認が必要 | targeted research で補強する |
| 仕様として読める | 後段の planner / architect へ渡す |

## 注意点

- **草案を勝手に完成扱いしない**: 未確定事項は明示して残します。
- **調査のために調査しない**: 仕様化に必要な論点だけを補います。
