# From Scratch

ゼロから仕様を立ち上げる sub-skill です。何を作るかがまだ曖昧な段階で、`spec-interviewer` を使って論点を掘り、必要に応じて `deep-researcher` へ渡します。

## こんなときに使う

このスキルは次のようなときに使います:
- 要件がまだ固まっていないとき
- 仕様の入口を会話から作りたいとき
- まず論点を整理してから調査したいとき

## ワークフロー: ゼロから仕様を立てる

### ステップ 1 — 目的と対象を掘る

`spec-interviewer` を使って、目的、対象範囲、制約、受け入れ条件の入口を揃えます。最初にここを揃える理由は、後で出る論点を比較しやすくするためです。

### ステップ 2 — 未確定事項を分ける

確認済み、推測、未確認を分けてメモします。曖昧さを残したまま進めると、仕様と仮説が混ざりやすくなるからです。

### ステップ 3 — 必要な根拠を調べる

論点の中で、事実確認が必要な箇所だけ `deep-researcher` に渡します。調査を広げすぎず、決定に必要な根拠だけを揃えます。

## 早見表

| 状況 | 次の動き |
| --- | --- |
| 目的が曖昧 | `spec-interviewer` で目的を絞る |
| 未確認が多い | 事実確認が必要な論点だけ `deep-researcher` へ渡す |
| 仕様の骨格が見えた | 後段の planner / architect へ渡す |

## 注意点

- **実装計画を作らない**: ここでの役割は仕様の入口づくりです。
- **調査を先に広げすぎない**: 先に論点を絞るほうが、後の調査が軽くなります。
