# Deep KPT reference

## 使う場面

- 問題が多かった
- 同じ問題が再発した
- 途中で戻りが発生した
- 原因が見えない
- 環境改善の種を強めたい

## KPT

- **Keep**: 続けたい良い動き
- **Problem**: 直したい詰まりや摩擦
- **Try**: 次に試すこと

### Try の扱い

Try は実験として書く。成功条件が見えるなら、T に落としてよい。

- Keep を伸ばす Try
- Problem を防ぐ Try
- 自動化・テンプレート化・導線改善の Try

## 深掘り

Problem が大きい場合は 5つのなぜで掘る。

1. 何が起きたか
2. なぜ起きたか
3. なぜそれが許されたか
4. どこで止められたか
5. どこを変えるべきか

その後、SMART に落とす。

## SMART

- Specific: 具体的に
- Measurable: 確認可能に
- Achievable: 実行可能に
- Relevant: 問題に直結させる
- Time-bound: 次回までに動ける形にする

## 環境改善ループ

ふりかえりで見えた改善種は、次の順で回す。

1. Session note に残す
2. T / TRY に入れる
3. 必要なら issue 化する
4. skill / hook / docs の修正に繋げる

このループにすると、個人の学びが環境の強化に変わる。
