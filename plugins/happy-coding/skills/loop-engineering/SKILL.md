# loop-engineering

この skill は、改善作業を 1 つの流れとして進めるための入口です。単発修正ではなく、失敗ログ・review 指摘・検証結果を整理しながら、次の実装や follow-up へ自然につなげます。

## こんなときに使う
- 失敗ログや review コメントを、原因確認・修正・検証・ふりかえりまでまとめて扱いたいとき
- 実装前に再現条件と成功条件を固定したいとき
- 実装後に確認結果を残し、必要なら後続 Issue 化や別 skill への引き継ぎをしたいとき
- 単発対応ではなく、改善ループを回したいとき

## ワークフロー
1. まず対象と成功条件を固定する
2. 再現条件、原因、影響範囲を確認する
3. 必要なら `design-and-plan` または `implement` に方針を渡す
4. 検証結果を確認し、必要なら `implementation-eval-gate` や `happy-add-issue` へ進める

## ルール
- 1 回の会話で 1 つの改善対象に絞る
- 実装に入る前に、再現条件と成功条件を明示する
- 検証の結果は次のアクションに残す

## 注意点
- すでに設計が固まっているなら `implement` へ直接進んでもよい
- 取りこぼしがあれば `happy-add-issue` で後続化する
