# PrivateEval

PrivateEval は、Loop Engineering の Evaluate で使う品質ゲートです。
AI が自分で作ったものを自分で褒めるためではなく、**機械判定で取れない品質を、明示した軸で点検し、次の Patch 対象を絞る**ために使います。

## 評価軸

| 軸 | 5 点に近い状態 | 低評価の兆候 |
|---|---|---|
| 構造化精度 | 問題、原因、影響範囲、変更範囲、検証方法が分かれている | 症状と原因を混同する。いきなり広範囲を書き換える |
| 形式知化力 | Why、判断理由、再現手順、制約が未来の自分やチームに残る | 動くが理由が分からない。PR / commit / README に判断が残らない |
| 完了条件設計力 | test / lint / typecheck / build / acceptance で完了を客観判定できる | 「動作確認しました」だけで終わる。失敗ケースが未確認 |
| 組織視点 | 保守性、安全性、レビューしやすさ、自走性に効く | その場しのぎ。自分しか分からない。将来の負債を増やす |
| 再利用性 | checklist、template、skill、reference として次回も使える | 今回限りで、同じ問題をまたゼロから解く |

## 採点基準

| 点数 | 判定 | 状態 |
|---:|---|---|
| 5 | 優秀 | チーム資産としてそのまま使える |
| 4 | 良い | 一部補足すれば十分使える |
| 3 | 普通 | 意図は分かるが改善が必要 |
| 2 | 弱い | 情報はあるが判断・共有に使いにくい |
| 1 | 不十分 | 成果物としての目的を満たしていない |

## 総合判定

| 総合点 | 判定 | 意味 |
|---:|---|---|
| 22〜25 | 優秀 | Happy AI Life の開発憲法に沿い、チーム資産として使える |
| 18〜21 | 良い | 実務投入可能。一部補強すれば安定運用できる |
| 13〜17 | 普通 | 動くが、構造化・検証・形式知化に改善が必要 |
| 8〜12 | 弱い | その場対応が強く、開発憲法とのズレが大きい |
| 5〜7 | 不十分 | 再設計が必要 |

## Critical 要件

次を満たさない場合は、点数が高く見えても Stop しません。

- エラーを握りつぶしていない
- 最小差分である
- test または客観的検証がある
- Why が残っている
- 無関係な変更がない
- セキュリティを悪化させていない

## Stop Condition

```text
必須:
- 機械判定が通っている
- Critical 要件がすべて OK
- セキュリティを悪化させていない
- 無関係な変更がない
- 最小差分で説明できる

PrivateEval:
- 総合点 22/25 以上
- 構造化精度 4 点以上
- 形式知化力 4 点以上
- 完了条件設計力 4 点以上

補助:
- 2 回連続で新しい重大指摘が出ない
- 3 ループして改善しない場合は Human Review へ切り替える
- 差分だけ増えて品質が上がらない場合は停止する
```

## 評価メモの最小形

低リスクな小タスクでは、毎回長い表を作らず次の形で足ります。

```markdown
PrivateEval:
- 落ちた軸: なし / <軸名>
- Stop 理由: <機械判定、Critical、PrivateEval の要点>
- 次回に残す型: なし / <README、template、skill、Issue など>
```

## 注意点

- PrivateEval は test / lint / typecheck / build の代替ではありません。
- 自己評価の点数より、どの軸を次に Patch するかを重視します。
- 低リスク作業では軽量化できますが、Critical 要件は省略しません。
- `skill-eval` は skill / prompt 資産の評価方法を扱います。PrivateEval は Loop Engineering の作業品質ゲートです。
