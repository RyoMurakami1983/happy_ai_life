# Session Story reference

## 目的

ふりかえりに入る前に、今回の流れを短い時系列で揃える。複雑な session ほど、いきなり YWT / KPT に入ると論点がずれやすい。

## 確認すること

1. 今回のゴールは何だったか
2. 途中で使った skill / hook / agent は何か
3. どこで出戻りや詰まりが起きたか
4. どの判断をやり直したか
5. 今回を Quick で閉じられるか、それとも KPT が必要か

## 聞き方のコツ

- 箇条書き 3〜6 行くらいで十分
- 事実を先に、評価を後に置く
- skill 名は実際に使ったものだけを挙げる
- 「なぜ戻ったか」が見えたら KPT に寄せる

## KPT に寄せるシグナル

- 複数 skill をまたいでいる
- ユーザーの指示で方針変更ややり直しが入った
- save 先、公開配慮、責務分離で迷いが出た
- 次回にも再発しそうな摩擦があった
