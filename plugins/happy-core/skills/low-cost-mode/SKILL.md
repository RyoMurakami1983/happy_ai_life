# Low-cost mode（低燃費モード）

この skill は、Matt Pocock さんの `caveman` を出発点に、GitHub Copilot CLI、日本語での対話、従量課金時代の省エネ運用へ合わせて再設計したものです。
目的は「雑に短くする」ではなく、**技術的な正確さを保ったまま、読む価値の低い token を削る**ことです。

## こんなときに使う

- 従量課金や premium request 消費を意識して、短い応答を既定にしたいとき
- 「短く」「省エネ」「低燃費」「要点だけ」とユーザーが求めたとき
- 調査・実装中の中間報告を、結論と次の動きだけにしたいとき
- 既に文脈が共有されており、長い再説明が不要なとき

## ワークフロー: 低燃費で答える

### Step 1 — 結論を先に置く

最初に結果、判断、または現在の状態を書きます。挨拶、相槌、依頼文の言い換えは削ります。

```text
低燃費: 原因は token expiry 条件です。`<` を `<=` に直します。
過剰: 承知しました。確認したところ、この問題はおそらく...
```

### Step 2 — 必要な情報だけ残す

削るもの:

- 長い前置き、相槌、一般論
- ユーザー依頼の再掲
- 既に共有済みの背景説明
- 目的に直結しない選択肢

残すもの:

- 正確な技術用語、path、command、error message
- 判断理由、リスク、安全確認
- 次の操作に必要な最小手順

### Step 3 — 例外では省略しすぎない

次の場面では低燃費を一時的に緩めます。

- 破壊的操作、secret、権限、外部公開など安全確認が必要
- 複数手順を間違えると危険
- 学習目的で、理由や段階説明が求められている
- ユーザーが「詳しく」「背景も」と求めている

## 出力パターン

```text
[結論]。[必要な理由]。[次の一手]。
```

例:

```text
未コミット変更あり。`low-cost-mode` と `session-handoff` 追加分だけです。focused validation を流します。
```

## 常時適用との関係

`home-template/.copilot/copilot-instructions.md` には、低燃費を既定にする短い常駐ルールだけを置きます。
この skill は、ユーザーが明示的に「もっと省エネに」「低燃費モードで」と求めたときの詳細ルールです。

## 注意点

- 短さのために曖昧にしない。
- エラー文、コード、コマンドは勝手に短縮しない。
- 安全確認や人間の判断が必要な場面では、必要な説明を省かない。
