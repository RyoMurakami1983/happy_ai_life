# 母艦の Copilot 環境リファクタ

## Executive Summary
- Copilot カスタマイズ環境を、home / repo / path-specific / skill / agent の責務で整理し直した。
- 一般原則は home に集約し、repo 側は repo 固有の事実と運用に寄せる形へ再配置した。
- 師範エージェントは改善提案既定へ寄せ、必要な場面だけ edit できる構造にして、過剰な越権を抑えた。

## Session Notes

### 主要な学び
- 一般原則を repo 側に散らすより、home に集約して repo は参照に徹した方が境界が明確になる。
- 調査系の細則は path-specific で広く持たせるより、repo の調査原則としてまとめた方が扱いやすい。
- 師範エージェントは「常に直す人」ではなく、「改善提案を返す人」にすると、判断責務が分離しやすい。
- 品質の横断原則は明示見出しにしておくと、他の LLM から見ても見落としにくい。

### 変更の要点
- home instructions に最小限の `## 品質` を追加した
- repo / repo-template の instructions を重複削減中心で整理した
- `research.instructions.md` を repo instructions に吸収して削除した
- 4 つの shihan agent を統一構成に簡素化した
- ADR を 2 本追加して、判断根拠を明文化した
