# .NET品質ゲートとレビュー基準の補強

## Executive Summary
- .NET の再現可能セットアップ導線に続き、quality gate と review checklist を target repo へ配布できる形まで整えた。
- `dotnet-quality.yml`、`dotnet-shihan` の checklist、`dotnet-setup-dev-environment` の Step 5 を揃え、`.editorconfig` と `global.json` を build contract として扱う方針を明確化した。
- さらに、`.slnx` 標準方針と例外運用の整合を見直し、例外時は `DOTNET_TARGET` で明示できるようにして skill と workflow の齟齬を解消した。

## Session Notes

### 主要な学び
- AI の自由生成を直接縛るより、`build` / `format` / `test` を通過条件として揃える方が再現性に効く。
- `.slnx` 標準化は強制一本化よりも、標準と例外の両方を明文化した方が現場運用に乗りやすい。
- setup skill、workflow、review agent の 3 層で同じ build contract を見ると、方針のぶれを早めに検出できる。

### 変更の要点
- `dotnet-setup-dev-environment` の Step 5 を、target repo 側 `.editorconfig` を育てる build contract 方針に更新した。
- `repo-template/.github/workflows/dotnet-quality.yml` を追加し、`global.json` 前提の restore / build / format / test gate と `DOTNET_TARGET` 例外経路を入れた。
- `dotnet-shihan.agent.md` に build contract、`.slnx`、bridge handoff、quality gate を含む checklist を追加した。

## Next Steps
- 必須の次フェーズはなく、残りは ADR 記述同期と repo-template 側の短い入口追記の 2 件だけ。
- この時点で .NET 再現性アークは、知識・gate・補助 automation の 3 層が一通りそろった状態になっている。
