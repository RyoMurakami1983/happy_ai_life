# .NET再現可能セットアップの標準化

## Executive Summary
- .NET 開発環境を AI 任せでも再現性高く進めるため、dotnet CLI を正道に据えた setup 導線を整備した。
- 新規作成は `.slnx` 推奨、既存 `.sln` の変更は `dotnet sln migrate` を標準にし、CLI-first の運用を明文化した。
- さらに、既存 repo 診断と `.NET Framework 4.x` / `netstandard2.0` / `.NET 5+` 共存への handoff を組み込み、greenfield 偏重を解消した。

## Session Notes

### 主要な学び
- 再現性を上げるには「生成ルール」より「通過条件」と「CLI-first の正道」を揃える方が効く。
- `.slnx` は単独の新形式として扱うだけでなく、既存 `.sln` からの移行手順まで含めて定義すると運用に乗せやすい。
- mixed TFM の現場では setup skill が全部を抱えず、bridge skill へ責務を渡す境界が重要だった。

### 変更の要点
- `dotnet-setup-dev-environment` skill を新設し、既存 repo 診断、`.slnx` 方針、bridge handoff、段階導入を追加した。
- `dotnet` router、`repo-onboarding`、C# instructions を更新し、既存 repo 起点でも同じ導線へ入れるようにした。
- build contract を母艦 root に置かず target repo 側に寄せる判断を ADR として明文化した。
