# SDD/DDD設計基盤と実装経験則の整備

## Executive Summary
- SDD の設計段に DDD の戦略・戦術を接続するため、design-workshop を router 化し、標準設計と Balanced Coupling レンズ設計を分岐できるようにした。
- さらに、サブドメイン分類から実装パターンとテスト方針を導く経験則を `_foundation` に追加し、設計判断の根拠を共有化した。
- 既存コードの結合構造を診断する modularity-review skill も追加し、設計と診断の導線を整理した。

## Session Notes

### 主要な学び
- 仕様、設計、実装を router / skill で分けると、フェーズ単位で差し替えやすい。
- DDD の戦略と戦術は別軸で整理したほうが、設計判断と実装判断が混ざりにくい。
- 共通参照を `_foundation` に置くと、ワークフロー間で用語と判断基準を揃えやすい。

### 変更の要点
- spec-interviewer / spec-workshop / design-workshop / SDD を連携する流れを確立した
- Balanced Coupling 参考資料、DDD 用語集、実装経験則を追加した
- design-workshop の router 化と modularity-review の追加を行った
- PR 作成、事前レビュー、ふりかえりの流れを一通り確認した
