# 師範 Agent 運用モデル（アーカイブ）

**日付**: 2026-04-07
**ステータス**: アーカイブ（Phase 1 で師範 agent は削除済み）

---

## 背景

4 つの師範 agent（dotnet-shihan, python-shihan, typescript-shihan, skill-shihan）は、現在「改善提案既定」を保ちつつ、Copilot CLI で有効な `edit` を frontmatter に含め、使用条件で実行権限を制御していました。

## Phase 1 での判断

2026-04 の agents/ ゼロベース見直しの Phase 1 において、`architect` / `planner` 以外のカスタムエージェントを削除しました。師範 agent はこの対象に含まれます。

言語固有の型判断・品質基準の提示・コードレビューなどは、今後 built-in 機能またはオーケストレーター自身で対応します。

## 保持した判断（アーカイブとして）

この ADR は、将来的に師範 agent を再導入する場合の参考資料として保持します。以下は削除時点での運用モデルです。

### 1. 3 要素の分離と段階適用

| 要素 | 判断 | 理由 |
|------|------|------|
| **(A) 本文簡素化** | 採用 | 統一構成で保守性向上。運用影響なし |
| **(B) 既定モード変更** | 条件付き採用 | 思考態度は求道者、行動権限は先生モードに分離 |
| **(C) edit 権限追加** | 条件付き採用 | 使用条件を厳格に定義した上で追加 |

### 2. 師範の責務境界（削除時点）

| agent | 責務 | 師範がやらないこと |
|-------|------|------------------|
| **師範（4 体）** | ドメイン固有の型・品質基準の提示、改善提案、コードレビュー | 構造判断の最終決定、計画立案、TDD の進行管理 |
| **architect** | 構造判断、トレードオフ分析、コンポーネント境界 | 言語固有のコーディング規約 |
| **planner** | 実装計画、段階分割、依存関係整理 | コードレビュー |

※ tdd-guide, refactor も Phase 1 で削除済み

### 3. 2 つのモードの定義（削除時点）

#### 改善提案既定（既定モード）

- **思考態度**: 常にカイゼンの目線で見る（パターンの限界を見極め、より良い型を探る）
- **行動**: レビュー・提案・判断理由の提示に留める
- **edit の使用**: しない（このモードでは）
- **出力**: 改善案と理由を提示し、ユーザーの判断を待つ

#### 求道者モード（明示要求時）

- **トリガー**: ユーザーの明示依頼（「求道者モードで」「このパターンを改善して」等）、または他 agent からの handoff
- **行動**: 最小編集で改善を実践する
- **edit の使用**: 許可
- **制約**: repo-wide の build/test/validation を守る、変更理由と影響範囲を事前に提示する

#### 先生モード（明示要求時）

- **トリガー**: 「先生モードで」「レビューだけして」「教えて」等
- **行動**: レビュー専用または学習支援専用
- **edit の使用**: しない
